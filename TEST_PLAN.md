## Test Files to Create

### 1. `tests/test_whatsapp_parser.py`
Unit tests for the WhatsApp chat export parser.

**Parse line patterns to test:**
- `"12/03/2025, 08:30 - Chidi: paid 5000"` → amount=5000, sender="Chidi"
- `"12/03/2025, 09:15 - Mary: 3000"` → amount=3000, sender="Mary"
- `"12/03/2025, 14:22 - Reghie: Mary 3000"` → amount=3000, sender="Mary", confidence=0.7 (uncertain)
- `"Nurse Ada: contributed N10,000"` → amount=10000, sender="Nurse Ada"
- `"Emeka 💰: transferred 5,000"` → amount=5000, sender="Emeka 💰"
- `"5000"` → amount=5000, sender=None, confidence=0.2, needs_review=True
- `""` (empty line) → skipped
- `"12/03/2025, 08:30 - "` → skipped (no sender)

**Amount detection edge cases:**
- `"5,000"` → 5000
- `"N5000"` → 5000
- `"N5,000.00"` → 5000 (strip kobo)
- `"5k"` → 5000
- `"five thousand"` → None, needs_review=True
- `"-5000"` → skip (invalid)
- `"N0"` → skip (invalid)

**File-level tests:**
- Empty file → returns []
- BOM character at start → handles gracefully
- Non-UTF8 encoding → raises encoding error
- 10,000+ line file → handles without timeout

### 2. `tests/test_payout_service.py`
Unit tests for the rotation logic.

**`get_next_recipient` test cases:**
- 5 members, no payouts yet → returns member with rotation_order=1
- Member with rotation_order=1 already paid in cycle 1 → returns rotation_order=2
- Last member (rotation_order=5) paid → returns None (cycle complete)
- Member skipped (rotation_order=3, not in payouts) → returns rotation_order=3
- 0 members → returns None

**`calculate_payout_amount` test cases:**
- 5 members paying N5000 each in cycle 1 → N25000
- 1 member with N0 contributions → N0 (edge case)
- No contributions recorded yet → N0

**`advance_cycle` test cases:**
- After last member pays, cycle should reset and increment

## Key Interactions to Verify
- WhatsApp message format: parser handles multiple variations
- Duplicate contribution warning: same member, date, amount
- Cycle completion: last member pays, cycle resets
- Reminder idempotency: cron fires twice in 15 min, member only gets 1 message

## Edge Cases
- Empty WhatsApp export → clear error message
- All messages unparseable → returns empty list with explanation
- Member pays before all others → payout records, cycle advances correctly
- Invalid phone number on member → reminder skipped, admin notified

## Critical Paths
1. WhatsApp export → parse → review → import → DB
2. Contribution recording → duplicate check → save
3. Payout recording → cycle advance → WhatsApp notification
4. Cron → check reminders due → send → log
